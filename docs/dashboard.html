EPSS Exploitation Probability

EPSS marginal cost on top of the model. Standalone EPSS at ≥ 0.50 fires on 44 events on this manifest; standalone EPSS at ≥ 0.10 fires on 90. NP+DI+DQ absorbs 43 of 44 (≥0.50) and 87 of 90 (≥0.10) before EPSS would have triggered — when an NP+DI rebuild rolls every dependency to latest, adjacent CVEs that have shipped a patch but haven't yet crossed an EPSS threshold get carried along incidentally. EPSS adds 1 marginal patch event on top of NP+DI+DQ at the 0.50 threshold and 3 at 0.10. EPSS isn't a competitor to the structure tests in well-run estates; it's a thin backstop for the floats.

Exploitation Rate by Stack Layer

Exploitation Rate (KEV / NVD)

KEV Distribution by Stack Layer

Key finding: OS/kernel CVEs have a 78% exploitation rate when time-windowed to 2021–2026 CVEs — the highest of any stack layer. Email/collaboration servers (100%) have a small NVD denominator that makes the rate an upper-bound artifact. Platform layers (OS, VPN, JVM, web servers) account for ~46% of windowed KEV entries, while libraries/frameworks represent less than 1% of KEV (4 entries). NVD undercounts library vulnerabilities by orders of magnitude — OSV tracks 770 critical/high across just 177 popular packages, with an estimated 93,000+ ecosystem-wide. The KEV library entries are almost all network parsers.
Note: … KEV entries for pre-2021 CVEs are excluded from exploitation rates to match the NVD 2021–2026 denominator window. These represent active exploitation of legacy vulnerabilities during the analysis period.

Critical/High Vulns per Package: Parser vs Non-Parser

Top 20 Libraries by Critical/High (OSV)

The fat tail: We queried 177 popular open-source packages across 6 ecosystems via OSV.dev. 62% of non-parser libraries have zero critical/high vulnerabilities. Network parsers are 2.5× more likely to accumulate C/H vulns (median 3 vs median 0). The top 10 packages hold 62% of all C/H advisories — 6 of 10 are network parsers. Outliers like TensorFlow (128 C/H) and Pillow (49 C/H) have high vuln counts but zero KEV entries. High vulnerability count alone doesn't predict exploitation; the network-parser signal does. Network Parsers as Exploitation Predictor

Network-Parsing vs Non-Parsing Exploitation Rate by CVSS

Exploitation Lift: Network-Parsing / Non-Parsing

Critical insight: Components that parse untrusted network traffic (HTTP requests, TLS handshakes, DNS responses, authentication tokens) are exploited at 3–6× the rate of non-parsing components at every CVSS severity level. A network-parsing HIGH (7.0–7.9) CVE has a higher exploitation rate than a non-parsing CRITICAL (9.0–9.9). "Does it parse untrusted network input?" is a better triage signal than CVSS score alone — especially for prioritizing library rebuilds.

Three Response Lanes

What the model fires versus what rides the cycle. Estate-maturity dimension (Cat 1 / 2 / 3 — who can absorb a patch) is independent and lives on the walkthrough §9.

Triggered — rebuild now

NP+DI in app layer or hacker S+A. 14 events/year → 2–6 after filter. Rebuild app + container base image.

Monthly cadence — cap blast radius

OS container refresh on a regular drumbeat. Reduces chain depth ~60-70% on the noise-cleaned LPE set. Not responding to specific CVEs — reducing attack surface.

Release cycle — ride it

Everything else — autobuild for open-source deps, BAU patch process for commercial / OS / appliance. No special action; the model didn't fire.

What this filter doesn't cover

The model targets the network-edge HTTP-parsing surface. Several CVE classes are deliberately out of scope — they have different exploitation dynamics, different threat models, or different patch-routing answers. Surfacing them here so a reader using the dashboard for triage knows what the filter doesn't claim.

Firmware/Hardware

3,739 NVD entries, 23 KEV (0.62%). Chipset BIOS, printer firmware, Qualcomm/Arm silicon — high barriers to weaponization, airgapped, update friction. Routed via vendor patch cycles, not the model.

Linux Kernel Local

Many criticals require local access. Attackers prefer remote-first vectors — local privesc is a second stage, not a first. Routed via OS patch cadence (per RHEL / Ubuntu errata).

Consumer IoT

Smart home, IP cameras, consumer routers — high CVSS but low strategic value for nation-state/criminal targeting. Different operational owner; not in this triage scope.

Non-Western Ecosystems

Products popular in specific regional markets (e.g., Huawei, Tenda) get NVD criticals but limited Western exploitation tracking. Coverage gap, not a model claim.

Memory Corruption

Developing a heap overflow exploit is hard, but once a PoC drops, weaponization follows in hours. The skill barrier filters discovery, not deployment. Worth watching at PoC publication, not at CVE.

Niche Products

Single-critical-CVE products with minimal deployment. Not worth exploit development for attackers. Watch list catches if deployment grows.
