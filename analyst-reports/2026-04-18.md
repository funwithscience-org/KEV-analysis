# KEV Analysis — Daily Analyst Report: 2026-04-18

## TL;DR

The KEV catalog has not been updated in two days (last release 2026-04-16; April total holds at 14), so the "new KEV entries" lens is quiet today — but the underlying CVE firehose is not. The data point that moves the ball: **CVE-2026-5194, a wolfSSL certificate-validation bypass affecting ~5 billion devices, is now publicly reported as having been discovered by Nicholas Carlini at Anthropic using Claude Mythos Preview (Project Glasswing).** That is the first wolfSSL CVE whose discovery chain is explicitly named as Mythos-originated in mainstream reporting — meaning the "40 attributed CVEs = maybe 40, maybe 0" ambiguity starts to resolve toward "at least a small but real number are bona fide AI-driven finds in Glasswing-target products." Separately, two critical (CVSS 9.0) Thymeleaf SSTI CVEs disclosed 2026-04-17 extend the library-framework / HTTP-parsing story and are natural KEV candidates if exploitation is observed.

### Most notable *non-KEV* CVEs disclosed in the last 24–48 hours

**CVE-2026-5194 — wolfSSL certificate-validation bypass (CVSS tbd, published ~2026-04-08 in v5.9.1 release notes; heavy press coverage 2026-04-13).** Glasswing-target product. Affects ECDSA/ECC, DSA, ML-DSA, Ed25519 and Ed448 signature paths — an attacker can present a forged cert with a digest shorter than cryptographically required, and wolfSSL accepts it. Scope is enormous: wolfSSL ships in routers, IoT, automotive, industrial, and military systems ("5 billion devices" per vendor marketing). Discovery is credited by wolfSSL to Nicholas Carlini (Anthropic). The Register / Hackread / SC Media / Cybernews are now reporting explicitly that Carlini used **Claude Mythos Preview** as the scanning tool. *This is the first mainstream-press-confirmed Mythos-in-production signal outside CVE-2026-4747 (FreeBSD NFS RCE).* Attribution nuance per VulnCheck: the 9 wolfSSL CVEs sit under a joint "Calif.io in collaboration with Claude and Anthropic Research" credit line — so wolfSSL is mixed attribution, not pure Glasswing. Still, CVE-2026-5194 is the cleanest Mythos-linked disclosure in a Glasswing-target product so far.

**CVE-2026-40477 and CVE-2026-40478 — Thymeleaf SSTI (CVSS 9.0, published 2026-04-17).** Two related server-side template injection bugs in Thymeleaf ≤ 3.1.3.RELEASE. Unauthenticated SSTI → RCE if user input reaches the template engine, which is the classic Spring-MVC-anti-pattern deployment. This is **library_framework** layer and **HTTP-parsing adjacent** (templates render HTTP responses). Pattern-wise these are Spring4Shell-rhyming — a template/expression engine restricted-object-scope bypass. No in-the-wild exploitation reported yet, but CVSS 9.0 and no-auth RCE mean KEV addition is plausible within days-to-weeks if operators start observing hits. These go on the watch list.

**CVE-2026-40258 — Gramps Web API path traversal (CVSS 9.1).** Authenticated, owner-privileged path traversal via malicious ZIP. Narrow impact (genealogy software), but fits the "arbitrary file write via archive extraction" pattern that periodically makes KEV (e.g., 7-Zip, WinRAR). Low probability of KEV but noting for completeness.

**CVE-2026-40352 — FastGPT NoSQL injection in password-change endpoint (CVSS 8.8).** AI-application-stack vulnerability. Auth bypass → account takeover. Interesting only because FastGPT is an AI tooling product; this is the kind of secondary-order CVE we should expect more of as AI-stack adoption grows (and which has nothing to do with AI *finding* bugs).

**CVE-2026-33825 (Microsoft Defender BlueHammer EoP)** — still not in KEV as of 2026-04-18, **11 days post-public POC**. This continues to be quiet data for the premise "public POC with working exploit rapidly drives KEV addition." Either CISA has not observed in-the-wild exploitation, or the EoP-on-local-admin threat model doesn't meet CISA's "actively exploited" bar, or both. Worth carrying forward.

## Mythos Signal Check

**Today's assessment: the Mythos signal has gone from "not visible in public data" (yesterday's framing) to "weak but now named in mainstream press," mostly on the strength of CVE-2026-5194 reporting maturing.**

What changed vs. 2026-04-17:
- **CVE-2026-5194 (wolfSSL) is now widely reported as a Mythos-Preview-assisted find.** Yesterday's report treated the 9 wolfSSL CVEs as inferred from researcher affiliation. As of today, at least one of the nine has explicit vendor-credited attribution to Nicholas Carlini (Anthropic) plus press-reported use of the Claude Mythos Preview tool. That's a meaningful de-risking of the "maybe 40, maybe 0" ambiguity Garrity/VulnCheck flagged.
- **Nuance: "Mythos-assisted" is not the same as "Glasswing-autonomous."** CVE-2026-4747 (FreeBSD NFS RCE) remains the only CVE Anthropic has publicly described as autonomously discovered and exploited by Glasswing/Mythos in the agentic sense. CVE-2026-5194 looks more like "a human expert using Mythos as an AI research tool," closer to the Bouncy Castle CVE-2026-5588 model. This is still a Glasswing-target product (wolfSSL) and still a meaningful production data point — but it's not proof of autonomous agentic discovery.

**Cumulative signal across the run (2 data points now):**
1. CVE-2026-4747 (FreeBSD NFS RCE) — autonomous Glasswing attribution per Anthropic.
2. CVE-2026-5194 (wolfSSL cert validation) — Claude Mythos Preview–assisted per press reporting; human researcher attribution.
3. Adjacent: CVE-2026-5588 (Bouncy Castle, Carlini using Claude, 2026-04-15) — Claude-assisted research, not Mythos-specific.

Three Claude-in-the-credit-line disclosures in under two weeks. All three are in exactly the product layers the Glasswing target list calls out (OS network stack, TLS library, crypto library). The pattern isn't noise anymore. It is also not yet a step-change — none are in KEV, so the *exploitation* half of the thesis is not yet tested.

**Devil's advocate, stated honestly:**
- Two of the three (Bouncy Castle, wolfSSL-5194) are Carlini-led research with Claude as a tool. That's "senior researcher productivity multiplier," not "AI system autonomously discovering bugs at scale." If the July 2026 Anthropic disclosure reveals that the 40 attributed CVEs are mostly of this flavor, the "Glasswing as step-function" narrative weakens — it becomes "Claude makes already-good researchers faster," which is real but not civilization-scale.
- wolfSSL has a long pre-Glasswing history of CVEs found by conventional means (Trail of Bits, Kudelski, NCC Group). A cert-validation bypass is exactly the kind of bug a determined human reviewer finds. Press-reported "Claude Mythos Preview use" may be marketing-shaped framing of what was a human-driven review.
- We do not have independent confirmation from wolfSSL (the vendor) of the AI-tool claim — that quote appears to come from Carlini or Anthropic communications, not wolfSSL's own advisory. Weight accordingly.
- Two wolfSSL CVEs on the same release (CVE-2026-5194 + CVE-2026-5501 in the OpenSSL compat layer affecting NGINX/haproxy integrations) do match the **cluster pattern** Mythos-style tooling would produce — multiple related findings disclosed together. That's stronger evidence than any one CVE on its own.

**Net:** yesterday I said "no visible Mythos signal." Today I'd say "one confirmed + one credibly reported Mythos-linked CVE in Glasswing-target products, with a clustering pattern emerging around wolfSSL." That is a small upgrade, not a paradigm shift. I want at least one more independent vendor-credited Mythos attribution before I'd move the rolling assessment further.

**Outside KEV, the disclosure firehose skewed toward library/framework today:** Thymeleaf ×2, wolfSSL ×2 (5194 + 5501), plus a long tail of application-layer CVEs. This is a reminder of the survivorship-bias thesis: libraries *are* getting exploitable bugs at pace; KEV just doesn't reflect library exploitation proportionally because exploitation telemetry attribution upstream-to-library is hard. Every Spring app pulling a vulnerable Thymeleaf is downstream-exploited as "the app," not as "Thymeleaf."

**HTTP-parsing thesis today:** holding to strengthening. Thymeleaf is a template engine that renders HTTP responses; wolfSSL 5501 affects NGINX/haproxy OpenSSL-compat integrations (directly HTTP-parsing). Neither in KEV yet, but the thesis's prediction about *where disclosable bugs cluster* is matched well.

**CWE families outside KEV (24h window):** CWE-94 (code injection) via Thymeleaf template execution, CWE-295 (cert validation) via wolfSSL, CWE-22 (path traversal) via Gramps, CWE-89 (SQLi/NoSQLi) via FastGPT. The 2026 pattern of injection-dominant CWEs in high-severity disclosures continues. No notable uptick in pure memory-corruption disclosures this cycle; if Mythos were primarily producing fuzzing-style memory bugs, we should be seeing more. We're not — which either means Mythos isn't primarily a fuzzer, or its fuzzing output hasn't hit the public feed yet.

## Big Picture

**What today tells us that yesterday didn't, in two sentences:** The Mythos-in-production signal went from "inferred from researcher affiliation" to "vendor-credited + press-reported in at least one Glasswing-target product." That is a small but real step — the kind of evidence that, repeated 2–3 more times over the next month, would force a revision of the "no inflection yet" posture.

**Is Glasswing changing the landscape yet?** Not in CVE volume (April extrapolating to ~6,077, below conservative anchor). Not in KEV additions (April at 14, pacing with 2026 average). Not in exploitation telemetry. But **is** in the attribution metadata of individual high-severity library CVEs — specifically in Firefox (28 credited), wolfSSL (9 credited, 1 now press-confirmed Mythos-assisted), and FreeBSD (1 autonomous). The signal is in *researcher credit lines and advisory language*, not in aggregate volume. That is exactly what you'd expect from a tool that produces targeted, high-quality findings rather than high-throughput noise — and it's also what you'd expect from a research program that is carefully staging coordinated disclosures through vendors. Both interpretations fit.

**The thing that would change my mind in either direction:**
- **Falsify (Glasswing isn't material):** July 2026 Anthropic disclosure turns out to be 40-ish CVEs mostly of the "Carlini-plus-Claude" flavor in mature-already-audited products (wolfSSL, Bouncy Castle, OpenSSL) with average severity no higher than industry baseline. In that world, Mythos is a real-but-incremental force multiplier for expert researchers, not a landscape-changing technology.
- **Confirm (Glasswing is material):** We see a May–June NVD lump of 5–15 CVEs across Firefox / wolfSSL / NGINX / FreeBSD / OpenSSL with coordinated publication timing, internal Anthropic discovery methodology attested in the advisories, and 20%+ landing in KEV within 30 days. That would be step-function evidence.

**CISO action this week vs. last week:**
- Same immediate-patch priority list: SharePoint (CVE-2026-32201), FortiClient EMS (×2), Ivanti EPMM, Acrobat. Unchanged.
- **New this week:** SBOM your wolfSSL footprint. If you have an IoT fleet, router infrastructure, or embedded Linux, assume some device uses wolfSSL < 5.9.1 and build a patch plan. CVE-2026-5194's scope ("5 billion devices") will exceed your asset inventory.
- **New this week:** audit Java applications for Thymeleaf 3.1.3.RELEASE or earlier; treat CVE-2026-40477/40478 as likely exploited-in-the-wild within 2–4 weeks. The Spring ecosystem's template-engine bugs tend to catalyze fast.
- Continue watching CVE-2026-33825 (Defender BlueHammer) — still no KEV add 11 days post-POC.

**What to watch next 7 days:**
1. Does Thymeleaf SSTI (40477/40478) enter KEV? That's the cleanest "library-with-CVSS-9-and-public-POC becomes KEV-listed" data point available.
2. Does wolfSSL CVE-2026-5194 attract observed-exploitation reports? It's a cert validation bypass requiring the ability to present a crafted cert; exploitation paths exist but aren't trivial.
3. Any additional Glasswing-target-product CVE with explicit Anthropic/Mythos attribution. The pattern is three in two weeks; if it becomes five in three weeks, the Mythos signal meaningfully hardens.
4. First NVD-published Firefox CVE with Anthropic credit line. Mozilla's disclosure cadence is typically monthly, so next window opens late April.
5. Does CISA add any of the backfill patterns from 2026-04-13 in more recent-CVE form? Sometimes CISA backfilling old entries correlates with fresh observed exploitation of the same bug-class.
