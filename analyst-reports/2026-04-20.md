**Four straight days with zero CISA KEV additions** (last release 2026-04-16, catalog version `2026.04.16`) — the longest quiet stretch of the month and almost certainly material evidence that the "public PoC / active exploitation ⇒ rapid KEV add" heuristic is slower than reputation suggests. No new Glasswing/Mythos-attributed CVEs surfaced in the last 24 hours; Anthropic-direct-credit count holds at 40 / 3 Claude-in-credit-line.

## New Entries of Interest

### Zero new KEV entries in the last 96 hours

CISA KEV `catalogVersion: 2026.04.16`, `dateReleased: 2026-04-16T17:01:07.1675Z` — verified directly this run. No additions on 2026-04-17, -18, -19, or -20. **April total holds firm at 14.** That's now the longest KEV dry spell of the year (previous longest: 4-day gap inside the Feb 2026 post-Patch-Tuesday week; we have now matched it). Inside a single month this isn't a crisis — KEV additions are CISA editorial decisions, driven by confirmed in-the-wild exploitation telemetry that CISA trusts and that the KEV team has cycles to vet. But **it's starting to weigh on two premises at once**: (a) "public PoC → KEV add in days" is clearly looser than the security-marketing machine suggests, and (b) if Mythos-era discoveries were already landing on CISA's desk via vendors, we would expect more, not fewer, April adds.

### Most notable disclosures in the last 24 hours

**CVE-2026-20180 — Cisco Identity Services Engine (ISE) RCE (CVSS 9.9, published 2026-04-15, press maturing 2026-04-19/20).** Cisco's security advisory describes multiple ISE RCE vulnerabilities exploitable via crafted HTTP requests. An authenticated attacker with **read-only admin credentials** (a low bar — those creds get phished, keylogged, and exfiltrated routinely) can obtain user-level code execution on the ISE node and elevate to root. In single-node deployments, successful exploitation produces DoS of the NAC plane: un-authenticated endpoints can't reach the network until the node recovers. Cisco says no in-the-wild exploitation yet. This is a **new watch-list add (server-side)**: HTTP-parsing-adjacent (crafted HTTP as attack vector), broad enterprise deployment (Cisco ISE is a de facto NAC standard in regulated verticals), clear high-exploitability profile, and the auth gate is read-only admin rather than root admin — the lightest plausible gate for a CVSS-9.9 RCE. The CVSS-9.9 rather than 10.0 is exactly the auth-requirement tax. On pattern grounds it's a reasonable candidate for KEV within weeks if even one ransomware crew lands valid read-only admin credentials at a Cisco customer.

**CVE-2026-33827 — Windows TCP/IP RCE (CVSS 9.8, April Patch Tuesday).** Wormable on systems with IPv6+IPSec enabled. This is a non-HTTP RCE and we deliberately do **not** add it to the watch list (scope boundary), but it is worth flagging: if this one ever reaches KEV, it's a potential first-class counter-example for the HTTP-parsing thesis — a pure network-stack RCE exploited at volume. We'll track it on the Thesis Challenge side of the ledger rather than on the watch list.

**CVE-2026-20204 — Splunk Enterprise low-priv → RCE (CVSS 7.1, published 2026-04-15).** Authenticated low-privilege user can upload to `$SPLUNK_HOME/var/run/splunk/apptemp` and achieve RCE through apptemp-directory handling. Splunk Web gating reduces blast radius. Not a watch-list candidate (auth gate + limited HTTP-parsing nexus), but another 2026 data point that Splunk continues to be a fertile audit target. Splunk has a steady CVE drumbeat in 2026; no KEV adds for Splunk this year yet.

**Horizon3.ai update on n8n Ni8mare (CVE-2026-21858)**, worth flagging separately. Horizon3's customer telemetry apparently found **no customer n8n instances impacted** even among those running vulnerable versions — because n8n in real-world enterprise deployment is typically internal-only (VPN / private network / cloud segment), not internet-exposed. This **softens but does not kill** the 100k-exposed-instance narrative from Cyera. Two reads: (a) the 100k figure includes a large population of self-hosted hobbyist instances that are not part of enterprise attack surface and that therefore won't generate KEV-worthy exploitation telemetry; (b) enterprise n8n deployment is mature enough that the KEV add we were forecasting in "weeks" may instead be "months or never." Noting this counter-data explicitly because it is the cleanest disconfirming evidence for our most confident recent prediction. I'm keeping CVE-2026-21858 on the watch list but moving its expected KEV probability **down**, not up.

## Mythos Signal Check

**Assessment essentially unchanged from 2026-04-19: weak but real attribution signal in Glasswing-target library layer; zero exploitation signal; no aggregate volume signal.** No new Mythos-attributed CVEs today. What the data has done over the last 72 hours is *not* accumulate more Mythos evidence — it has accumulated more evidence that the stories getting *exploited* this cycle are not Mythos-adjacent. If there is an AI-mediated security story this week, it is the same as last week's: **AI-ecosystem products** (Marimo, n8n, FastGPT, Hugging Face as delivery channel) are the ones actively getting exploited, not Glasswing-target libraries.

Three Claude-in-credit-line disclosures in two weeks, all in product layers the Glasswing target list calls out (OS network stack, TLS library, crypto library). None in KEV. **The exploitation half of the thesis remains completely untested.** This is not the moment to upgrade the signal — today produced no new direct-attribution data points — but it is also not the moment to discount it.

**Devil's advocate — today's sharper framing.** A reasonable counter-read is: **Claude-in-the-credit-line is becoming the more relevant attribution population than Anthropic-direct Glasswing discovery.** CVE-2026-34197 (Apache ActiveMQ, KEV-confirmed) was traced by Naveen Sunkavally at Horizon3.ai using Claude as an analysis tool in ~10 minutes. If we count "Claude-assisted exploitation research that ends up on KEV," the ActiveMQ entry is arguably the single most important Claude-mediated security data point of April 2026 — because it's the one that actually hit CISA's threshold for active exploitation. That's a different story than "AI discovers CVEs at scale"; it's "AI accelerates the analyze-and-weaponize step of the exploitation pipeline." Both can be true, and both matter — but they have different policy implications, and the July disclosure narrative will be misleading if it conflates them.

## Thesis Challenge (Non-HTTP Library Exploits)

**No new counter-examples today.** Searched across April 2026 KEV additions (still 14, unchanged since 2026-04-16) and the 24h non-KEV disclosure window; **no library or framework CVE has reached KEV via a purely non-HTTP vector this cycle.** Today's closest near-misses:

- **CVE-2026-33827 (Windows TCP/IP RCE, CVSS 9.8)** is a pure network-stack RCE and not HTTP-parsing. If this one lands on KEV, it will be the single strongest counter-example we have seen this year. Windows itself remains scope-excluded (OS, not library), but if the exploitation pattern spreads into third-party network stacks (e.g., lwIP, VxWorks), that becomes a credible disconfirming pathway. Not yet.
- **CVE-2026-20204 (Splunk Enterprise)** is authenticated file-write-then-RCE, which can be reached via HTTP (Splunk Web) or via direct filesystem access (apptemp). The HTTP path is the practical vector for remote attackers, so it does not disconfirm.
- **CVE-2026-5588 (Bouncy Castle)** still has only TLS-adjacent exploitation paths discussed publicly — HTTP-parsing-adjacent.

The absence of counter-examples continues to be weakly *for* the thesis while acknowledging the detection-bias problem: non-HTTP library exploitation (file-parsing libs via email attachments, crypto libs via stored credentials, serialization libs via message queues) is structurally hard to observe and attribute, and so would be systematically undercounted in our data. No table updates this run.

## Big Picture

**What today tells us that yesterday didn't, in one paragraph.** The KEV dry spell has hit 96 hours and counting — the longest quiet stretch of April, matching the longest of the year. Two candidates from this week (Marimo with 662 confirmed exploit events and n8n with public PoC + CVSS 10.0) are *not* being added. That's a meaningful piece of mechanism-level disconfirming evidence for a premise the industry takes for granted: the path from "public PoC + observed in-the-wild" to "CISA KEV entry" is apparently weeks-to-never, not hours-to-days. For the HTTP-parsing thesis and the time-to-exploit-compression thesis, this doesn't falsify anything — those theses are about exploitation, not about CISA editorial velocity. But it does rebuke any simple "watch-list entry hits 100% of observed exploitation" framing. The watch list is measuring the **KEV editorial threshold**, not the **exploitation threshold**, and those two are looking increasingly distinct.

**Is Glasswing/Mythos changing the vulnerability landscape yet?** Still no at the aggregate level. April extrapolates to **5,547** (down from 5,793 yesterday, down from 6,082 two days ago, down from 7,000+ early-cycle conservative projection). KEV at 14/month pacing ~42% below the 2026 monthly baseline. Zero Glasswing-target products in April KEV. W16 — the first full post-launch week — came in as the slowest of three. Four-day KEV dry spell. This is not what "AI inflection" looks like; this is a quiet month. The NVD-latency argument still protects Glasswing through mid-May, at which point its credibility fades fast.

**Is AI adoption changing the landscape yet?** Yes, but as last week — via the attack-surface-expansion channel, not the discovery channel. Marimo, n8n, FastGPT and the Hugging Face delivery vector are today's actual AI-security story, and a CISO who hasn't asset-inventoried those is strictly worse off than one who has. That story is now 9 days old and keeps adding beats (NKAbuse via Hugging Face; Ni8mare PoC release; 100k exposure number though Horizon3 softens it).

**If I were a CISO this week vs. last week:**
1. **New this week (day 20):** patch Cisco ISE immediately (CVE-2026-20180). Read-only admin credentials get phished; if you have ISE internet-exposed or reachable from any segment an attacker can pivot to, you have a P1 within the next few weeks once exploitation catches up. Broad enterprise deployment + CVSS 9.9 + HTTP attack vector is a combination that historically doesn't age well.
2. Carry forward from last week: Marimo/n8n asset inventory for data-science teams; SharePoint (32201), FortiClient EMS (×2), Ivanti EPMM, Acrobat (34621).
3. **Continue to de-prioritize** relative to where I started the month: BlueHammer (33825) — 13 days post-PoC, no KEV, actual exploitation seems modest.
4. **Windows TCP/IP (CVE-2026-33827)**, wormable on IPv6+IPSec — patch via April Patch Tuesday; the wormable-RCE framing makes this the single most contingently-scary Patch-Tuesday fix of the month even if exploitation is not yet observed.

**What to watch, next 7 days:**
1. Does CISA KEV break the silence? A 5-day gap would be the longest of 2026. The longer this goes, the more we should suspect (a) CISA editorial capacity constraints, or (b) a structural rate-limit on KEV adds that is independent of disclosure volume. Either has real implications for the watch-list design.
2. Does CVE-2026-39987 (Marimo) enter KEV? Forecast: **50/50 within 7 days** (down from yesterday's "most likely single data point"), given today's reinforced evidence that observed-exploitation-but-no-U.S.-victim-telemetry may not clear CISA's threshold.
3. Does CVE-2026-20180 (Cisco ISE) attract any reported exploitation? Cisco ISE KEV entries have historically moved within 4–6 weeks of first in-the-wild observation; if we see any Shadowserver or GreyNoise exploitation telemetry, we'd expect KEV within another 2–3 weeks after that.
4. Does the first post-Glasswing-launch Firefox or wolfSSL CVE with explicit Anthropic attribution land? Plausible window opens next week given NVD publish latency.
5. April NVD total — end-of-month extrapolation at 5,547 is well inside the "no Glasswing surge" band; final-week-surge is still possible but shrinking as a probability.
