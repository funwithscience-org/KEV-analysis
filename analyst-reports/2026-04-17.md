# KEV Analysis — Daily Analyst Report: 2026-04-17

## TL;DR
April 2026 KEV additions (14 through day 17) are pacing with the 2026 monthly average and skew heavily Microsoft/Fortinet/Ivanti — classic enterprise-edge exploitation, not AI-discovery fingerprints. The actively exploited SharePoint zero-day (CVE-2026-32201, CVSS 6.5) is this week's strongest data point against "CVSS predicts exploitation," while the absence of Glasswing-target products in KEV continues to support the NVD-publish-latency hypothesis rather than any Glasswing inflection. Ten days post-Mythos-Preview launch, the exploitation landscape is still dominated by pre-Glasswing patterns.

## New Entries of Interest

Ordered by date added, with stack-layer and thesis-fit annotations.

**CVE-2026-34197 — Apache ActiveMQ Improper Input Validation (added 2026-04-16, CWE-20/CWE-94).** Middleware/message-broker layer. Input-validation → code injection is the exact pattern that drove the 2023–2024 ActiveMQ exploitation wave (CVE-2023-46604). HTTP-parsing adjacent (ActiveMQ's OpenWire and STOMP listeners are protocol parsers). This is a recurrence of the same bug class in the same product — a data point for "library/middleware survivorship bias *eventually* resolves via repeated exploitation."

**CVE-2026-32201 — Microsoft SharePoint Server Improper Input Validation (added 2026-04-14, CWE-20, CVSS 6.5).** This is the headline entry. It's the actively exploited zero-day from April Patch Tuesday, spoofing over the network, added to KEV the same day the patch shipped. Two signals here:
- Time-from-disclosure-to-KEV ≈ 0 days. That's not a Glasswing artifact; that's CISA's now-standard behavior for in-the-wild 0-days.
- CVSS 6.5 ("Medium") on an actively exploited server-side HTTP-parsing bug. Strong counter-evidence that CVSS tracks exploitation. Every quarter we find more of these.

**CVE-2026-21643 — Fortinet FortiClient EMS SQL Injection (added 2026-04-13, CWE-89).** VPN/network-appliance management plane. Classic unauthenticated SQLi in an admin surface — this is the 2024 FortiClient EMS bug rhyming in a new way. HTTP-parsing adjacent (the EMS web admin is the attack surface). Fortinet now has two KEV entries in one week (also CVE-2026-35616, improper access control, added 2026-04-06).

**CVE-2026-34621 — Adobe Acrobat/Reader Prototype Pollution (added 2026-04-13, CWE-1321).** Prototype pollution in a document reader is unusual enough to note. This fits the "client-side document parser" pattern and is an odd candidate for manual discovery — prototype-pollution chains in JS-heavy client apps are exactly the kind of deep-in-the-parser bug a fuzzer-with-semantic-reasoning would surface. *Not* evidence for Mythos specifically; just flagging it.

**CVE-2026-1340 — Ivanti Endpoint Manager Mobile Code Injection (added 2026-04-08, CWE-94).** Another enterprise-mobility/MDM management-plane unauth RCE. HTTP-parsing adjacent. Ivanti has been a KEV regular since 2023; this maintains the pattern.

**CVE-2026-5281 — Google Dawn Use-After-Free (added 2026-04-01, CWE-416).** Browser rendering (WebGPU implementation). Genuinely interesting because Dawn is one of the components Glasswing-adjacent targets — complex, parser-heavy C++ in the browser pipeline. UAF via crafted HTML is Mythos-plausible, but Google's internal fuzzing (OSS-Fuzz, Libfuzzer, ClusterFuzz) is the baseline explanation and more than sufficient. Play-devil's-advocate: Google has been finding Dawn UAFs continuously since 2023.

**Backfill noise:** Four older CVEs were added 2026-04-13 (CVE-2009-0238 Office, CVE-2012-1854 VBA, CVE-2020-9715 Acrobat, CVE-2023-21529 Exchange, CVE-2023-36424 Windows OOB read). These are CISA catching up on observed historical exploitation. They distort any "time-to-KEV compression" metric if counted naively — must be filtered out for trend work.

## Mythos Signal Check

**Honest answer: no visible Mythos signal in today's KEV additions.** None of the 14 April KEV entries are in Glasswing-target products (Firefox, wolfSSL, F5 NGINX Plus, FreeBSD, OpenSSL). The previously confirmed Glasswing-attributable CVE count remains at 40 (28 Firefox, 9 wolfSSL, 1 each F5/FreeBSD/OpenSSL), with only CVE-2026-4747 (FreeBSD NFS RCE) explicitly attributed by Anthropic.

The one adjacent signal worth naming: **CVE-2026-5588 (Bouncy Castle, published 2026-04-15)**, which Nicholas Carlini disclosed was found using Claude. That's Claude-assisted but *not* Project Glasswing and *not* in KEV. It's a useful data point for "Claude-as-research-tool" but not for "Mythos Preview as autonomous discovery system in the wild."

**Cumulative read across the short run we've been doing:** The NVD publishing latency argument is holding up. NVD typical publish lag is 4–8 weeks; Mythos Preview launched 2026-04-07, so the earliest window where Glasswing discoveries could reasonably appear in the public NVD feed is late April at the earliest, with meaningful volume in May–June. April CVE volume extrapolating to ~6,242 (below the 6,500 conservative anchor, only marginally above March's 6,304) is *consistent with* that latency argument — but it is also consistent with "Glasswing's exploitation-relevant discovery count is smaller than the marketing suggests." We cannot distinguish those yet. That is the honest position.

**Devil's advocate on our own flagging:** If we were looking for confirmation bias, Dawn UAF and Acrobat prototype pollution both *could* be squinted into "AI-discovered" narratives. Both have perfectly adequate conventional explanations (Google OSS-Fuzz, Adobe's own fuzzing program). Absent attribution, we should not code them as Mythos signal.

**HTTP-parsing-adjacent entries this month: ~7 of 14 (50%)** — SharePoint, Exchange, Fortinet (x2), Ivanti, ActiveMQ (OpenWire/STOMP), NetScaler. The 3–6x HTTP-parsing lift thesis holds at the top-of-funnel. If anything, April is over-indexed on management-plane HTTP admin surfaces (Fortinet EMS twice, Ivanti EPMM, SharePoint, NetScaler) — the "edge device with a web admin" pattern is the 2026 story so far.

**CWE family distribution:**
- Input validation / injection (CWE-20, CWE-94, CWE-89, CWE-1321): 6 entries — dominant
- Memory corruption (CWE-416, CWE-125): 3 entries

- Auth/access (CWE-284, CWE-59, CWE-426, CWE-494): 4 entries
- Injection continues to outpace memory-corruption in KEV adds, which runs counter to the naive "AI fuzzing → more memory-corruption in KEV" prediction. If Mythos were landing, we'd expect the memory-corruption share to rise. It hasn't — yet.

**New product families vs. 2021–2024 baseline:** Nothing novel. Fortinet EMS, Ivanti EPMM, SharePoint, Exchange, ActiveMQ, NetScaler are all well-established KEV regulars. The most novel entry is Google Dawn (browser WebGPU), which is genuinely new territory but one CVE is noise.

## Big Picture

**What today's data tells us that yesterday's didn't:** Not much that's novel — and that's the story. Ten days after the Mythos Preview launch and Project Glasswing announcement, the KEV feed and the April Patch Tuesday both look completely normal. Microsoft's 163/167-CVE April Patch Tuesday is large but not historically anomalous (October 2025 was 167, pre-Glasswing). The SharePoint 0-day is a classic server-side spoofing bug that reads like a conventional discovery. The two Fortinet EMS entries in one week read like a coordinated vendor patch cycle, not an AI firehose.

**Is Glasswing changing the landscape yet? Not observably in public data.** The most honest framing: we are inside the NVD publish-latency window where Glasswing-attributable CVEs *could not yet* appear in volume. That window closes roughly mid-May. If by 2026-05-15 the CVE volume, KEV additions, and Glasswing-target product CVE counts all still look like trailing 2026 behavior, the "Glasswing as step-function" narrative becomes materially weaker. If instead we see a May/June lump in Firefox/wolfSSL/NGINX/FreeBSD/OpenSSL advisories and a matching uptick in coordinated-disclosure clusters, the thesis is vindicated.

**CVE volume interpretation:** March 2026's 6,304 was a genuine YoY jump (+57% vs. March 2025's 4,012) and predates Glasswing by weeks. April is pacing at 6,242 extrapolated — meaning the March spike appears to be the new normal, not an outlier that was going to regress. The 2026 Q1 run-rate (~5,400/mo) is structurally higher than 2025 (~4,000/mo) for reasons unrelated to AI discovery: CNA expansion, MITRE triage backlog clearance, and broader disclosure norms. Any Glasswing signal will have to be extracted *against* this already-elevated baseline, which makes detection harder.

**CISO action this week vs. last week:** Same priorities — patch the April Patch Tuesday SharePoint 0-day (CVE-2026-32201) immediately; patch the two FortiClient EMS entries; patch Ivanti EPMM; confirm Acrobat is updated across fleet. The *new* thing this week: treat the Bouncy Castle CVE-2026-5588 as an early signal that crypto-library bug classes may surface in ways your asset inventory doesn't cover. Many organizations don't have a Bouncy Castle SBOM line item; they should.

**Watch for in next 7 days:**
1. Does CVE-2026-33825 (Defender BlueHammer EoP) get added to KEV? If so, that's evidence the "public POC drives KEV addition" heuristic is faster than exploitation-in-the-wild signal.
2. Any follow-on CVEs in Apache ActiveMQ or Fortinet EMS — these vendors often get disclosed in clusters.
3. First appearance of a Firefox / wolfSSL / NGINX / FreeBSD / OpenSSL CVE that could be Glasswing-attributable in the public NVD feed. Mid-to-late April is the earliest plausible window.
4. Any advisory where the credit line names "Mythos Preview," "Claude," or "automated AI analysis" as the discoverer. Carlini/Bouncy Castle is the template.
